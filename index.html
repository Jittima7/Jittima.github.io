เปรียบเทียบความเหมือนและความแตกต่างในการเขียนโปรแกรมแบบ OOP ด้วยภาษา Java, C++, Python

ความเหมือน

ทั้งสามภาษาสนับสนุนการเขียนโปรแกรมเชิงวัตถุ (OOP) โดยมีหลักการดังต่อไปนี้:

  • การใช้ Class และ Object: ทั้งสามภาษาสามารถสร้างคลาสและอ็อบเจ็กต์ได้ เช่นการใช้คลาส Animal และสร้างอ็อบเจ็กต์จากคลาสเหล่านั้น
  • การสืบทอด (Inheritance): สามารถสืบทอดคลาสได้ เช่น Dog และ Cat สืบทอดจากคลาส Animal
  • การพ้องรูป (Polymorphism): ทั้งสามภาษาสามารถ override method ในคลาสลูก เช่น การ override method sound() ใน Dog และ Cat
  • Destructor: ทั้งสามภาษามี destructor เพื่อทำลายอ็อบเจ็กต์ เช่น delete animal, delete dog, และ delete cat

ความแตกต่าง

ถึงแม้หลักการ OOP จะเหมือนกัน แต่มีข้อแตกต่างบางประการในรายละเอียด:

  • Java: Java มี garbage collector ที่จัดการการทำลายอ็อบเจ็กต์โดยอัตโนมัติ การสร้างและทำลายอ็อบเจ็กต์ใช้ keyword new และ finalize() ในการทำลาย
  • C++: C++ ต้องจัดการหน่วยความจำเองผ่าน new และ delete และสนับสนุนการ overload operator รวมถึงมีความยืดหยุ่นสูงในเรื่องการจัดการหน่วยความจำ
  • Python: Python ใช้ garbage collection เช่นเดียวกับ Java แต่การเขียนโปรแกรมจะกระชับกว่า และการสร้างและทำลายอ็อบเจ็กต์จะจัดการเองผ่านการเรียก __init__() และ
