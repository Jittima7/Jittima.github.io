เปรียบเทียบความเหมือนและความแตกต่างในการเขียนโปรแกรมแบบ OOP ด้วยภาษา Java, C++, Python

1. ความเหมือนในการเขียนโปรแกรมแบบ OOP

ทั้งสามภาษานี้ (Java, C++, Python) รองรับหลักการเขียนโปรแกรมเชิงวัตถุ (OOP) โดยมีโครงสร้างพื้นฐานที่คล้ายกัน:

  • สามารถสร้าง Class และ Object ได้
  • รองรับ Inheritance หรือการสืบทอด
  • สามารถใช้ Polymorphism เพื่อให้ Method ที่ชื่อเดียวกันทำงานต่างกันในคลาสที่แตกต่างกัน
  • มีการจัดการ Constructor และ Destructor เพื่อสร้างและลบอ็อบเจ็กต์

2. ความแตกต่างในการเขียนโปรแกรมแบบ OOP

Java

ในภาษา Java คลาสทั้งหมดจะถูกห่อหุ้มอยู่ในคลาสใหญ่หนึ่งคลาสเสมอ และไม่จำเป็นต้องจัดการการลบวัตถุ (Destructor) เนื่องจาก Java มีระบบ Garbage Collection ที่จะจัดการเรื่องการคืนหน่วยความจำอัตโนมัติ

class Animal {
Animal() { System.out.println("create animal"); }
void sound() { System.out.println("animal makes a sound"); }
}

C++

ใน C++ เราจำเป็นต้องลบวัตถุที่สร้างขึ้นเองด้วยคำสั่ง delete เนื่องจาก C++ ไม่มี Garbage Collection และมีการใช้ pointer ในการจัดการหน่วยความจำ

class Animal {
public:
Animal() { cout << "create animal" << endl; }
virtual void sound() { cout << "animal makes a sound" << endl; }
virtual ~Animal() { cout << "delete animal" << endl; }
};

Python

Python มีโครงสร้างที่ง่ายกว่าและสามารถลบวัตถุได้อัตโนมัติเช่นเดียวกับ Java โดยใช้ระบบ Garbage Collection และใช้การย่อมาจาก __del__ สำหรับ Destructor

class Animal:
def __init__(self):
print("create animal")
def sound(self):
print("animal makes a sound")
def __del__(self):
print("delete animal")
